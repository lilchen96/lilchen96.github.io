浅谈cookie
发表于 2019-09-27| 更新于 2020-03-31|技术
阅读量:

基本信息

  cookie 是一个非常具体的东西，指的就是浏览器里面能永久存储的一种数据。跟服务器没啥关系，仅仅是浏览器实现的一种数据存储功能。
  cookie 存储在用户本地终端。
  cookie 大小限制一般为 4k。
  cookie 会增加请求大小。
  cookie 是存储在本地的，可以找到相应的文件。

如何设置和发送 cookie

  服务器在 response head 中可以设置 set-cookie，来存储 cookie 到客户端。
  浏览器发送请求时，request head 中可以携带 cookie。

会话期 cookie 和 持久化 cookie

  没设置 Expires 或 Max-Age 的 cookie 为会话期 cookie，即结束会话（关闭浏览器）就被清除。会话期 cookie 存储在浏览器内存中。但浏览器通常支持会话恢复功能。
  设置了 Expires 或 Max-Age 的 cookie 为持久化 cookie，经过指定时间后失效。

SameSite 新属性

  Chrome 51 开始，浏览器的 Cookie 新增加了一个 SameSite 属性，用来防止 CSRF 攻击和用户追踪。
可以设置三个值Strict，Lax，None。
  Strict: Strict 最为严格，完全禁止第三方 Cookie，跨站点时，任何情况下都不会发送 Cookie。换言之，只有当前网页的 URL 与请求目标一致，才会带上 Cookie。

  Lax: 规则稍稍放宽，大多数情况也是不发送第三方 Cookie，但是导航到目标网址的 Get 请求除外，包括三种情况：链接，预加载请求，GET 表单。

  None: 网站可以选择显式关闭 SameSite 属性，将其设为 None。不过，前提是必须同时设置 Secure 属性（Cookie 只能通过 HTTPS 协议发送），否则无效。

cookie 其他属性

  cokie-name=cookie-value: cookie 以 key value 键值对形式设置。

  Domain: 指定 cookie 可以送达的主机名。假如没有指定，那么默认值为当前文档访问地址中的主机部分（但是不包含子域名）。

  Path: 指定一个 URL 路径，这个路径必须出现在要请求的资源的路径中才可以发送 Cookie 首部。

  HttpOnly: 设置了 HttpOnly 属性的 cookie 不能使用 JavaScript 经由 Document.cookie 属性、XMLHttpRequest 和 Request APIs 进行访问，以防范跨站脚本攻击（XSS）。

  Secure: 设置后 cookie 只能通过 HTTPS 协议发送。

cookie 应用场景

  例如自动登录，获取用户喜好。较为隐私或涉及安全问题的数据不应该存储在 cookie 中，除非进行加密处理。

cookie 中的 sessionId

  cookie 中会存储 sessionId，服务器会根据 sessionId 获取 session 中的数据，session 即相当于 cookie 在服务器端的扩展。一是因为 cookie 存储空间小，二是因为隐私信息存储在 cookie 中不安全。
