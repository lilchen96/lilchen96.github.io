记一次浏览器兼容问题
发表于 2020-03-26| 更新于 2020-03-31|技术

起因

  别人用 360 浏览器，发现 select 框展示不出来，如图：

调查

  上网搜索后发现 360 浏览器有两个模式，极速模式是 chrome 内核，而兼容模式是 IE 内核。经验证，切换到极速模式，在 chrome 内核下不会产生该问题。那么罪魁祸首就是IE 内核的兼容问题了。
  但是项目中已经使用了 babel 了。为什么还会有兼容问题呢？
  打开控制台，发现是 iview 的 select 组件用到了Array.prototype.includes()方法报错了。去 MDN Web Docs 上看了下这个 API，果然是 IE 不支持。转译新 API，就需要安装使用babel-polyfill了。

babel 和 polyfill 的区别

  babel 默认只转换新的 JavaScript 句法（syntax）。
  babel-polyfill 是转换新的API，用于实现浏览器并不支持的原生 API 的代码。
  由此可见，JavaScript 兼容问题得从两个维度去解决：句法兼容和API 兼容。
